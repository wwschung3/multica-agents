When writing/editing a ticket:
1. **When Writing Ticket Title**: Use plain text without markdown formatting
2. **When Writing Ticket Description**:
   - Use actual newlines (press Enter in shell)
   - Do NOT use literal \n in the command
   - MUST NOT use \\n as literal text instead of actual line breaks
   - Do NOT wrap text in ** for bold
   - Structure with proper markdown headers (#, ##, ###)

## Key Rules
- Use actual line breaks in command (not \n literal)
- Avoid bold markers in description
- Use markdown headers (##) for sections
- Use hyphen (-) for bullet points
